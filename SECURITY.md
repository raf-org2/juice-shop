# Security Policy

## Reporting a Security Issue

We take the security of our project seriously. If you believe you have found a security vulnerability, we encourage you to report it to us as quickly as possible.

**Please DO NOT report security issues via public GitHub issues.**

Instead, please report security vulnerabilities privately via [your preferred contact method, e.g., email address, security platform, or dedicated form].

Example: security@example.com

## Vulnerability Handling

Upon receiving a security report, our team will:

1.  **Acknowledge Receipt:** We will acknowledge your report within [e.g., 2 business days].
2.  **Investigation:** We will investigate the reported vulnerability to confirm its validity and impact.
3.  **Remediation:** If confirmed, we will work on a fix and prioritize its release.
4.  **Disclosure:** We aim to disclose vulnerabilities responsibly, coordinating with the reporter and potentially releasing a security advisory or update notes.
